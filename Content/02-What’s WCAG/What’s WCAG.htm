What's WCAG

This section tells all you should know about accessibility basics and gives useful links to dig deeper.

Web Content Accessibility Guidelines (WCAG) is a technical standard created by the Web Accessibility Initiative (WAI) of the World Wide Web Consortium (W3C) in cooperation with individuals and organizations worldwide. Its main goal is to provide a single shared standard for web content accessibility that meets the needs of every user, no matter if it is an individual, organization, or government.

WCAG was primarily created for:

  • Web content developers (page authors, site designers, etc.).

  • Web authoring tool developers.

  • Web accessibility evaluation tool developers.

  • Others who want or need a standard for web accessibility, including mobile accessibility.

     

What’s the newest version?

WCAG 2.0 was published on 11 December 2008 (approved as an ISO standard: ISO/IEC 40500:2012. ISO/IEC 40500)

WCAG 2.1 was published on 5 June 2018 (the latest standard).

WCAG 2.2 is to be finished in September 2022 (draft version).

Should I cover all of them?

If you want to meet the latest accessibility standards, you can use the 2.1 resources; no need to start from the lowest standard number. Let me assure you that according to W3C, WCAG 2.1 does not replace or cancel previous WCAG standards. At the same time, W3C encourages you to use the most recent version available when developing or updating your content.

The WCAG standards have 12-13 guidelines organized under 4 principles: perceivable, operable, understandable, and robust.

Credits: Punchkick Interactive

If you want to check more details, testable success criteria for each guideline can be met at three levels (from lowest to highest): A, AA, and AAA.

They show if your page meets the criteria and its conformance to WCAG.

 

What if you’re not a developer or technical writer, but want to learn more about accessibility standards for your job? No need to worry. There are many WAI Resources more useful for researchers, policymakers, or managers.
