Images

This section provides information about images / screenshots used in documentation, and why you should always remember about ALT texts.

Using images or screenshots is important in technical documentation. It allows you to show users the exact place you refer to in documentation.

When creating accessible images / screenshots, remember to:

  • Think about the screenshot’s strategic purpose (no need for decorative ones).

  • Stick to statics formats; no animations or flashy GIFs are needed.

  • Use vector images (SVG) or images with resolution that will stay sharp when enlarged (e.g., a 1000px wide file for a 500px wide placement).

  • If the text in the image is longer than a sentence, reproduce it on the page as regular text (e.g., essential information from a table or chart)

  • Logos, charts, or tables can have text on them as long as the alternative text is available.

    • For each image, provide a caption, Alt attribute (Alt text), title attribute, or aria-label attribute.

      Best practices for Alt texts:

      • Use "real" text. Alt text added to the image should describe what the image is about.

      • It should be clear, concise, and helpful.

      • Use up to 4 sentences.

      • Do not use “image of,” “screenshot of,” “photo of”.

      • If you use an image as a link, remember that the link's destination should be stated in the ALT.

      • If there is some text on the image, it is a good practice to duplicate it.

      • Make sure that ALT texts of images used for controls, status indicators, or repeated programmatic elements are consistent.

      Examples:

      CORRECT:

      Credits:18F: Digital service delivery

      Credits:18F: Digital service delivery

      NOT CORRECT

      Credits:18F: Digital service delivery

      Credits:18F: Digital service delivery

  • Do not use images for text. Re-create its text and attach an image next to it (PDF is not mandatory).

Test for accessibility

Use tools

To check images on a page, use Images Bookmarklet for Accessibility Testing or an automated testing tool, for example, WAVE.

or

Use the web developer tool

  1. Press Control + Shift + J (Windows, Linux, Chrome OS) or Command + Option + J (for Mac) to access the web developer tool.

  2. Click select images > Display Alt Attributes & Outline All Images.

  3. Check every ALT text according to “Best practices.”

or

Use inspect element tool

  1. Right-click outlined images and select Inspect element

  2. Check every ALT text according to “Best practices.”

 

Interested in Applicable WCAG Success Criteria for images? Check SC 1.4.5 Images of Text
