CSS

This section shows why CSS is no good for accessibility.

CSS is, by default, NOT ACCESSIBLE.

Screen readers ignore pages that are built around CSS. There are two main problems with assistive technology and CSS:

  • Content order. CSS can create different content orders than the ones used by screen readers, which causes chaos for users.

  • Images added via CSS do not have Alt attributes and are ignored by CSS. Therefore, adding <img> tags to all images uploaded via CSS and treating them as decorative is recommended.

     

Best practices for working with CSS:

  • Do not use CSS to create meaningful content

  • Use CSS ONLY to change or supplement the design

  • Do not use CSS to change the semantics of HTML

  • Do not size your content to width:0px, height:0px. It will not be visible

  • Make sure that any ordering and positioning defined in styles reflects the DOM tree and the reading order of your documentation

  • Use <.scr-only> (screen reader only) CSS class. It allows you to hide content on the page, but screen readers will have no problem accessing it. Remember to also reference it from within the tag of the element being hidden, for example, <div class="sr-only">We decide that this is hidden</div>

  • Make sure your CSS does not mess with the default accessible style

  • Adapt CSS to devise capabilities and user performance

  • Do not use <display: none> or <visibility: hidden>; (they hide information from screen readers), unless you want to hide content for all of us

  • But do not use <hidden> attribute

  • Make sure to add “You are here” before the breadcrumbs to make it easier for assistive software to read

  • Use skip links. To learn more, read the Links section

 

Testing for accessibility:

  1. Disable CSS

  2. Look for missing images and text

  3. Check if content order makes sense without CSS

  4. Look for overlapping text

  5. If you cannot see any weird elements, ask developers for help

  6. If possible, work with developers and look for code or other items they hide
