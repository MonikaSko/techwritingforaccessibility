Identifying language

This section provides information on how and why you should identify content in other languages in your documentation.

Most technical documentation is created in English and localized for other languages.

When creating technical documentation and preparing examples of selected functionalities in different languages, it is good to add this information to the code or use a specific button to detect such a text. Of course, this will not have a specific meaning for users without disabilities, but it will be a game-changer for assistive software.

How to identify a language?

  • You can add a language attribute to the HTML:

    Copy 
    <p>The book title is "<span lang="fr">Le Bon Usage</span>"

     

    or

    Credits: w3.org

 

If you do not add a <lang> attribute to your code, screen readers will pronounce all your French examples on the page as if it was written in English. The result can be confusing and hard to understand.

  • Most content management systems (CMS) have a button for such a case.

    To use it, you should:

    1. highlight the content they need to verify,

    2. click the button,

    3. select the correct language from the list.

      Credits: w3.org

Remember to use the <dir> attribute when working with bidirectional texts, such as Arabic, Hebrew, etc.

If the language on the page you want to discover has no markup around it, use <span>, <bdi>, or <div> attributes around its content. For example:

Copy 
<p>In Chinese we would call it <span lang="zh-Hans">一个挑战</span>.</p>

 

Test for accessibility

Use tools

To check part of the text and identify its language, you can use, for example, the WAVE tool to scan the page. It will identify a language tag with a globe icon.

Credits: Princeton University

 

or

Use manual testing

  1. Open the Report inspector that lists all the objects contained in a report in a tree structure.

  2. Locate the main HTML tag and check if the <lang> attribute is set to the correct language (different from the default one).

  3. If not, change it to the correct language.

If you leave any <lang> attribute with no language identified, it will be impossible for screen readers to identify and read it correctly.

 

To learn more about applicable WCAG Success Criteria, go to SC 3.1.2 Language of Parts
