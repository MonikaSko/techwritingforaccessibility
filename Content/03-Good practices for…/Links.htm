Links

This section provides information about creating accessible links for and in the documentation.

Most users skim the page for links, so your link titles should be informative and simple. If links are well created, assistive software users can press a hotkey to generate a list of links on the page and jump between them. Remember that all such links are generated out of context, so their correct naming is crucial for screen readers. You can use call-to-action or keywords to make your links more SEO-friendly.

How to prepare well-structured links?

  • Use links that clearly describe what the link/page is about

  • Use meaningful keywords if possible

  • Underline or mark links with color

  • Make sure your link is well readable on its own, without additional context

  • Avoid full URLs. Screen readers will read them letter by letter

  • Avoid “click here” or “read more” as a link copy

  • Avoid many links leading to the same content/page

  • If your link leads to the PDF document, specify its type and size (for example, TECH WRITING EXCUSES (PDF, 6MB))

  • Try to use your links as call-to-actions if possible

  • Consider adding the “skip navigation” link or “temporarily hidden skip link” at the top of your page. Such a link is helpful for users operating a keyboard with motor disabilities. It uses a negative value tabindex=”-1” and allows users to jump straight to the main content by only pressing the TAB button. It also allows the user to bypass potentially inaccessible content, such as complex tables, or social media feeds.

Screenshot from Starbucks.com

There is no need for multiple “skip links” on the page, no matter how developed it is.

 

Testing for accessibility:

Check links manually

  1. Check the page for full links displayed on the page. Provide full text for them.

  2. Verify links with no actual content like “Read more” or “Click here”. Check them for title attributes and modify them accordingly to the topic.

  3. Find all links with the same destination and try unifying them or deleting some of them if possible (the fewer differently named links to the same page, the better).

  4. Identify links with the identical copy. Check them for title attributes and modify them accordingly to the topic.

  5. Look for links that open in a new window with target='_blank'. Check if their indication is given accessibly.
