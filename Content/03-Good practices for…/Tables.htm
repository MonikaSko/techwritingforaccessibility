Tables

This section provides basic information about creating accessible tables.

Tables are a natural part of most technical documentation. They provide nice and clear data presentation and can be used in various ways. But are they accessible? They can be.

Best practices for tables:

  • Define column and row headers.

  • Do not merge cells; screen readers can't correctly read them.

  • Align tables with the text.

  • Add programmatic code equivalent for all the attributes of the visual formatting.

  • Add a caption and summary of the table's content and purpose above the table. It can be helpful when read by the screen reader.

  • Do not use tables for layout. They show 2-dimensional relationships between header and data cells. When used as a part of the layout, they can confuse assistive technology users. They will hear a lot of information about tags related to columns and rows that are usually a part of tables and content division in them. Additionally, the content will not be very clear for tools that help zoom pages.

  • Mark header cells with <th> and data cells with <td>.

  • For more complex headers <scope>, <id>, and <headers> attributes may be needed.

The more table types, the merrier...

Depending on the type of table you want to create, you have to add different attributes to make it accessible. You can either ask your friendly developer for help or check the examples below:

  • Tables with top row headers only.

  • Table with two headers.

  • Table with irregular headers.

  • Tables with multi-level headers.

If you use Content Management Systems (CMS) or Web Authoring Tools, check if they can help you with defining, for example, header cells. Many of them have appropriate functions added so that you do not have to work with code manually.

 

Testing for accessibility

Use tools

To look for lists and check if they are well-formatted into tables, you can use:

  • Tables bookmarklet

  • Accessible Web Helper

  • WAVE

or

Use manual testing

Check if:

  • All header cells have <th> for appropriate columns and rows

  • All data cells have <td> for appropriate columns and rows

  • There is a summary of the table placed under the table

  • Page is not using tables as part of the layout

 

Check applicable WCAG Success Criteria SC 1.3.1 Info and Relationships.
